# 🚀 Linux CI/CD Pipeline Automation

An advanced **DevOps-integrated** automation project designed to simulate a full Continuous Integration and Continuous Deployment (CI/CD) lifecycle using Docker containers and Bash scripting.

---

## 🛠️ Tech Stack & Tools

### 💻 Operating Systems
*   🟡 **LINUX** (Base OS)
*   ⚪ **DEBIAN TRIXIE** (Build Server & Pipeline Environment)
*   🔴 **CENTOS 7** (Target Production Server)

### ☁️ Virtualization & Containers
*   🔵 **DOCKER** (Containerization Engine)

### ⚙️ Automation & Version Control
*   📜 **BASH SCRIPTING** (Pipeline Logic)
*   📂 **GIT** (Version Control System)

---

## 📂 Project Structure
*   **deploy.sh**: The core automation script for testing and deployment.
*   **Dockerfile**: Configuration for the Debian build environment.
*   **README.md**: Project documentation and tech stack overview.

---

## 🚀 How it Works
1.  **Code Integrity**: The `deploy.sh` script starts by validating the environment.
2.  **Build Phase**: Compiles and prepares artifacts within the Debian container.
3.  **Deployment**: Simulates the automated transfer of verified code to the production container.
